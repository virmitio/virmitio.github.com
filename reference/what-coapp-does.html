1) How does the software get installed?

Indirect method

The first and most common way that CoApp packages will be installed is as dependencies of another package. To clarify, it is expected that the average user will not be looking specifically for zlib, or openssl, or any other development or runtime library directly. Rather, we expect that most users will go looking for end-products such as Apache, PHP, or MySQL. Downloading and running the installers for these "top-level" products will automatically download and install any packages that are needed to run them. In this way, many packages will be consumed (downloaded and installed) almost exclusively by other packages. A major benefit to this is that the end-user never needs to worry if the version of Apache or PHP they're installing has all of the latest patches built it. (More on this later in this document.)

Direct method

Suppose for a moment that you are a developer instead of an end-user, though. How do you install a package so you can build with it? Well that's no great effort either. Developer libraries are available in packages as well to simplify building programs. All that's needed is to locate the package you need (zlib-dev, for an example), and install it like any other program.
